# 增值税发票会计分录工作流优化建议

## 1. 开始节点配置优化

### 输入变量配置：

```json
{
  "AGENT_USER_INP": {
    "type": "String",
    "description": "用户本轮对话输入内容",
    "required": false
  },
  "input_img": {
    "type": "Image",
    "description": "增值税发票图片",
    "required": true
  },
  "user_role": {
    "type": "String",
    "description": "用户身份：销售方/购买方",
    "required": true,
    "enum": ["销售方", "购买方"]
  }
}
```

## 2. OCR节点配置优化

### 输入参数映射：

- `file_url`: 引用 `input_img`
- `page_start`: 1
- `page_end`: 1

### 输出变量处理：

- `code`: 状态码
- `data`: OCR识别结果数组
- `message`: 处理消息

## 3. 大模型节点配置优化

### 系统提示词模板：

```
你是一个专业的会计助手，专门处理增值税发票的会计分录生成。

## 会计处理规则：

### 销售方会计分录：
1. 确认销售收入：
   - 借：应收账款/银行存款
   - 贷：主营业务收入
   - 贷：应交税费-应交增值税(销项税额)

2. 结转成本（如有）：
   - 借：主营业务成本
   - 贷：库存商品

### 购买方会计分录：
1. 确认采购成本：
   - 借：原材料/库存商品/固定资产等
   - 借：应交税费-应交增值税(进项税额)
   - 贷：应付账款/银行存款

## 输出要求：
1. 根据用户身份（销售方/购买方）生成对应的会计分录
2. 分录格式：借：科目名称 金额，贷：科目名称 金额
3. 金额计算要准确，包含税额处理
4. 提供简要的会计处理说明

## 输入信息：
- 发票信息：{{ocr_data}}
- 用户身份：{{user_role}}
```

### 用户提示词模板：

```
请根据以下增值税发票信息，以{{user_role}}的身份生成相应的会计分录：

发票信息：{{ocr_data}}

请按照标准会计格式输出分录，并说明处理依据。
```

### 输出变量配置：

- `REASONING_CC`: 模型思考过程
- `accounting_entries`: 会计分录
- `explanation`: 处理说明

## 4. 结束节点配置优化

```json
  "invoice_info": "{{ocr_data}}",
  "accounting_entries": "{{accounting_entries}}",

}
```

## 5. 工作流测试建议

### 测试用例：

1. **销售方测试**：上传销售方开具的发票，验证销售收入确认分录
2. **购买方测试**：上传购买方收到的发票，验证采购成本确认分录
3. **特殊发票测试**：测试不同税率、多商品明细的发票处理

### 预期输出示例：

#### 销售方分录：

```
借：应收账款 11,300.00
贷：主营业务收入 10,000.00
贷：应交税费-应交增值税(销项税额) 1,300.00
```

#### 购买方分录：

```
借：原材料 10,000.00
借：应交税费-应交增值税(进项税额) 1,300.00
贷：应付账款 11,300.00
```

## 6. 错误处理建议

1. **OCR识别失败**：提供手动输入发票信息的备选方案
2. **发票格式异常**：提供常见发票字段的说明和手动校正功能
3. **金额计算错误**：添加金额验证和重新计算功能
4. **身份选择错误**：允许用户重新选择身份并重新生成分录

## 7. 性能优化建议

1. **缓存机制**：对相同发票的OCR结果进行缓存
2. **批量处理**：支持多张发票的批量处理
3. **模板管理**：为不同类型的发票建立处理模板
4. **历史记录**：保存处理历史，支持查询和导出
